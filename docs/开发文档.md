## 1. 项目概述

### 1.1 一句话定义

**OhMyCopy** 是面向家庭局域网（≤10 台）的轻量级跨设备剪贴板同步工具：Windows / Linux 原生客户端，对等 TCP 同步，UDP 广播发现，共享密码鉴权，egui 现代化 UI，追求**小体积、低内存、高可靠**。

### 1.2 背景与动机

| 痛点 | 说明 |
|------|------|
| 远程办公 | 通过 Sunshine + Moonlight 连接服务器进行办公与 AI 创作，频繁跨设备复制提示词、图片、文件 |
| 现有方案不可靠 | CrossPaste 等工具存在「复制后无法粘贴、必须重启」类故障，官方修复节奏不可控 |
| 资源占用过高 | Java 方案安装包数百 MB、RAM 占用大、UI 在 N100 等低功耗 CPU 上响应慢 |
| 体积预期 | 作为剪贴板同步工具，**安装/分发体积以 10MB 为上限目标** |

### 1.3 成功标准（验收 KPI）

| 指标 | 目标 |
|------|------|
| 可靠性 | 连续使用 ≥7 天无需因同步故障重启进程 |
| 延迟 | 同局域网文本：复制后 **≤1s** 对端可粘贴 |
| 体积 | Release 单文件 / 安装包 **≤10MB**（strip + LTO） |
| 内存 | 空闲常驻 RSS 目标 **≤50MB**（N100 可接受） |
| CPU | 空闲接近 0；事件驱动，避免高频轮询 |
| 平台 | Windows 10/11 + Linux（含 Debian） |

### 1.4 非目标（明确不做）

- 公网中继、NAT 穿透、云账号体系
- 移动端（iOS / Android）首期不做
- 浏览器扩展 / WebView UI
- 完整「网盘级」大文件传输器
- 与 Moonlight/Sunshine 剪贴板通道深度耦合
- 自动选主、复杂分布式共识
- M1 不引入 PAKE（SPAKE2/CPace 等）；家用共享密码模型足够
- 密码变更自动广播到其他设备（改密后由用户在各设备手动更新）

### 1.5 术语约定

| 用语 | 使用场景 |
|------|----------|
| **设备** | 用户界面、文档叙述（「附近设备」「已连接设备」） |
| **节点** | 协议与实现（`device_id` 标识的网络节点） |
| **载荷** | 帧/事件中的内容字节（不再混用 payload 作正文术语；代码标识符可用 `payload`） |
| **对端** | 已建立会话的另一个节点 |

## 2. 用户与场景

### 2.1 目标用户

- 家庭多 PC 用户（约 6 台，含 Debian）
- 远程桌面办公 + AI 创作（提示词、截图、小文件频繁互拷）
- 对体积、内存、稳定性敏感，可接受功能精简

### 2.2 核心场景

1. **本机 ↔ 远程服务器**：本机复制提示词 → 服务器侧立即粘贴到 AI 工具  
2. **服务器 → 本机**：服务器复制结果/日志 → 本机粘贴  
3. **图片同步**：截图 / 参考图跨设备粘贴（受大小上限约束）  
4. **小文件**：配置范围内的文件随剪贴板同步；超限提示，不静默失败  
5. **多机同时在线**：客厅 / 书房 / 迷你主机等任意子集互相同步  

### 2.3 设备与规模

| 项 | 约束 |
|----|------|
| 规模 | ≤ 10 台 |
| 网络 | 仅局域网 |
| 拓扑 | 对等（P2P），无强制中心服务器 |
| 系统 | Windows、Linux（Debian 等） |

## 3. 产品功能

### 3.1 功能优先级

| 优先级 | 功能 | 说明 |
|--------|------|------|
| **P0** | 剪贴板监听与写入 | 文本为主；可靠、可恢复 |
| **P0** | 对等 TCP 同步 | 每节点监听；**每对节点恰好一条连接** |
| **P0** | 密码鉴权 + 载荷加密 | 共享密码；禁止明文传密 |
| **P0** | 防回环 / 去重 | `event_id` 已见集强制；同步写入不外发 |
| **P0** | 断线重连 + 监听自愈 | 指数退避 + 随机抖动；杜绝必须整进程重启 |
| **P0** | 可配置内容大小上限 | 超限本地提示，不发送 |
| **P0** | UDP 广播设备发现 | 自动列出局域网内节点 |
| **P0** | egui 现代化 UI | 设置、设备列表、状态、基础历史 |
| **P0** | 防火墙引导 | 首次监听失败时提示放行端口 |
| **P1** | 图片同步 | 规范化为 PNG/JPEG 字节 + MIME |
| **P1** | 文件同步（小文件） | 超限提示；可配置 |
| **P1** | 本地历史记录 | 搜索、一键再次复制 |
| **P1** | 系统托盘 | 开机可启、状态图标 |
| **P2** | 置顶 / 收藏条目 | 常用提示词 |
| **P2** | 仅发送 / 仅接收模式 | 降低双向冲突场景 |
| **P2** | 内容类型过滤 | 如不同步密码管理器格式 |

### 3.2 功能说明

#### 3.2.1 同步内容类型

| 类型 | 行为 |
|------|------|
| 纯文本 / Unicode 文本 | 默认同步，最高优先级保障；线上规范为 UTF-8 |
| 图片 | 同步规范编码字节 + MIME（见 §6.4） |
| 文件 | 小文件整包传输；超过 `max_payload_bytes` 则 UI/托盘提示 |
| 其他格式 | 首期可忽略或降级为「有数据但未支持」提示 |

#### 3.2.2 大小限制

- 全局配置项：`max_payload_bytes`（**默认 10MiB**，可改）
- 可选分项：文本 / 图片 / 文件各自上限（后续）
- 超限：本地明确提示（原因 + 当前限制），**不发送、不半传**

#### 3.2.3 设备发现（UDP 广播）

- 节点周期性或启动时发送发现广播
- 广播载荷：**不包含密码明文**；含协议魔数、版本、节点 ID、显示名、TCP 端口、能力标志
- 收到广播后展示在「附近设备」列表；用户确认或自动加入（可配置）
- 支持手动添加 IP:Port 作为发现失败时的回退
- 可选 `goodbye` 宣告，加速 UI 离线展示（对端仍以超时为准）

#### 3.2.4 身份验证

- 全网共享密码（家庭场景）
- 连接建立后挑战-应答；密码 + **协议固定 salt** 派生密钥
- 后续帧均加密；错误密码无法完成握手
- **改密**：用户在每台设备上手动更新同一密码；不提供自动传播

## 4. 系统架构

### 4.1 总体架构

```text
┌─────────────────────────────────────────────────────────┐
│                     OhMyCopy 进程                        │
│  ┌─────────────┐  ┌──────────────┐  ┌────────────────┐  │
│  │  egui UI    │  │  同步引擎     │  │  剪贴板适配层   │  │
│  │  托盘/窗口  │←→│  Engine      │←→│  Win / Linux   │  │
│  └─────────────┘  └──────┬───────┘  └────────────────┘  │
│                          │                               │
│           ┌──────────────┼──────────────┐               │
│           ▼              ▼              ▼               │
│    ┌────────────┐ ┌────────────┐ ┌────────────┐        │
│    │ TCP 会话   │ │ UDP 发现   │ │ 本地存储   │        │
│    │ 单连接/对  │ │ 广播/监听  │ │ 配置/历史  │        │
│    └────────────┘ └────────────┘ └────────────┘        │
└─────────────────────────────────────────────────────────┘
         │ TCP 数据面              │ UDP 发现面
         ▼                         ▼
    其他设备 :port            255.255.255.255 / 子网广播
```

### 4.2 拓扑：对等（P2P）

**不设强制中心服务器。** 每一台运行中的 OhMyCopy：

1. **始终监听** TCP 端口（接受入站连接）
2. **按 tie-break 规则** 决定是否主动连接某一对端
3. 同一二进制、同一种运行模式

| 角色 | 说明 |
|------|------|
| 软件角色 | 每节点对等，无 master 进程 |
| 运维角色 | 常开 Debian 可作「事实上更稳定的节点」，但非特殊服务端 |

全互联：10 台最多约 **45 条** TCP 连接（每对一条），完全可接受。

### 4.3 TCP 连接模型（强制）

**任意两个节点之间最多保留一条 TCP 连接。**

#### 4.3.1 主动连接方（tie-break）

双方均已知对方 `device_id`（UUID）与地址后：

```text
若 local_device_id < remote_device_id（字节序比较）：
    本节点负责 dial（主动连接）
否则：
    本节点只 accept，不向该对端 dial
```

- 若同时出现双连接（竞态）：**保留「主动方 = 较小 device_id」那条**，关闭另一条。
- 连接表主键：`remote_device_id`（不是 socket 地址），IP 变化时迁移会话。

#### 4.3.2 重连

- 指数退避 + **全抖动**（例如 `base * 2^n + random(0, base)`），避免网络恢复时连接风暴
- 仅 **dial 职责方** 发起重连；accept 方等待入站

### 4.4 模块划分

| 模块 | 职责 |
|------|------|
| `clipboard` | 监听变化、读写；平台 → 规范内容映射 |
| `engine` | 已见集、防回环、限流、大小校验、路由到已连接节点 |
| `net::tcp` | 单连接管理、tie-break、帧编解码、心跳、重连 |
| `net::discover` | UDP 广播发送/接收、设备表维护 |
| `auth` | Argon2id、握手、会话密钥、AEAD 帧 |
| `store` | 配置、已知设备、历史记录 |
| `ui` | egui 界面、托盘、通知 |
| `app` | 生命周期、tokio↔UI 桥接、优雅退出 |

### 4.5 进程内通信与 egui–tokio 桥接（M0 锁定）

**选定模式：双通道 + 请求重绘**

| 方向 | 机制 |
|------|------|
| 引擎 → UI | `tokio::sync::mpsc` 或 `std::sync::mpsc` 推送 `UiEvent`；UI 每帧 `try_recv`，并 `ctx.request_repaint()` / `request_repaint_after` |
| UI → 引擎 | `tokio::sync::mpsc` 发送 `EngineCommand`（改配置、暂停同步、手动添加设备等） |
| 共享只读快照 | 可选 `arc-swap` / `RwLock` 存放「连接列表、最近错误」供 UI 即时读 |

约束：

- **egui 仅在 UI 线程调用**；剪贴板平台 API 若要求特定线程，由 `clipboard` 模块封装，不阻塞 UI 过久
- 单一 `tokio` 运行时在后台线程；`eframe` 主循环在主线程
- 关闭主窗口时可隐藏到托盘；退出时：发 shutdown → 等网络任务 ≤N 秒 → 超时强制退出

## 5. 网络与协议

### 5.1 传输分层

| 层 | 技术 | 用途 |
|----|------|------|
| 发现面 | **UDP 广播** | 设备发现、在线宣告 |
| 数据面 | **TCP** | 剪贴板内容、可靠有序传输 |
| 安全 | 密码 + 固定 salt 派生密钥 + 会话 AEAD | 鉴权与载荷保护 |

说明：

- 不做 WebSocket
- 首期不上 QUIC
- 公网不在产品范围内

### 5.2 UDP 发现协议

**默认端口**（可配置）：`UDP 3721`

**广播地址**：`255.255.255.255` 或子网定向广播；`SO_REUSEADDR` 按需。

#### 5.2.1 发现包结构（逻辑）

```text
magic:     "OMCP" (4 bytes)
version:   u16
msg_type:  u8   // 1=announce, 2=goodbye
device_id: 16 bytes UUID
tcp_port:  u16
name_len:  u8
name:      UTF-8（显示名）
caps:      u32  // 能力位：text / image / file ...
```

> v0.2：**移除** `fingerprint` 字段（对称密钥模型下无非对称身份可指纹化）。若日后需要展示「配置一致性」，可另加可选 `config_hash`（非安全边界）。

#### 5.2.2 行为

- 启动后立即 announce，之后每 3–10s 一次（可配置，**带抖动**）
- 忽略本机 `device_id` 的回环包
- 超过 N 秒未再收到 announce → UI 标为离线
- **禁止**在 UDP 中传输密码或剪贴板正文

### 5.3 TCP 数据协议

**默认端口**（可配置）：`TCP 3721`（可与 UDP 同号）

#### 5.3.1 连接生命周期

```text
（dial 方）TCP connect  /  （accept 方）accept
  → 交换 device_id + 协议版本（明文 Hello 最小集或首帧）
  → 双连接检测与裁撤（§4.3）
  → Auth 握手（挑战应答 + 派生 session_key）
  → 加密帧会话
  → 心跳 keepalive
  → 优雅 close / 异常后由 dial 方退避重连
```

#### 5.3.2 帧格式

逻辑帧经 **postcard** 序列化后，由 **ChaCha20-Poly1305**（或等价 AEAD）封装上线。

明文逻辑结构示意：

```text
len:     u32   // 密文或 body 长度（实现时定义外层信封）
type:    u16
flags:   u16
body:    bytes // postcard 编码的类型化消息
```

| type | 名称 | 说明 |
|------|------|------|
| 1 | `Hello` | 版本、device_id |
| 2 | `AuthChallenge` / `AuthResponse` | 握手 |
| 3 | `ClipboardEvent` | 剪贴板条目 |
| 4 | `Ack` | 可选 |
| 5 | `Ping` / `Pong` | 心跳 |
| 6 | `Error` | 错误码与信息 |

#### 5.3.3 ClipboardEvent body（逻辑）

```text
event_id:     UUID          // 强制；全局唯一（发送方生成）
source_id:    device_id     // 原始产生该内容的节点
content_hash: 32 bytes      // BLAKE3（辅助校验/展示，非主去重键）
kind:         text | image | file
mime:         字符串        // 如 text/plain; charset=utf-8、image/png
created_at:   u64           // unix ms
file_name:    可选字符串    // kind=file 时
payload_len:  u32
payload:      bytes         // 规范载荷（见 §6.4）
```

MVP：单帧载荷 ≤ `max_payload_bytes`；后续可扩展分块。

### 5.4 鉴权与加密（已锁定）

| 项 | 决策 |
|----|------|
| 模型 | 全网 **同一共享密码** |
| Salt | **协议级固定 salt**（常量，写入源码/规格，所有节点相同）。**禁止**每设备随机且互不同步的 salt |
| KDF | `Argon2id(password, FIXED_SALT) → root_key`；参数兼顾 N100；**进程内缓存 root_key** |
| 握手 | nonce 挑战 + 证明持有 root_key（HMAC 或 AEAD 密封） |
| 会话 | 派生 `session_key`；业务帧 AEAD 加密 |
| PAKE | **M1–M4 不做** SPAKE2/CPace |
| 失败 | 断开；UI 提示鉴权失败 |

> 说明：固定 salt 使 Argon2 主要起「拉伸密码」作用，而非抗全局彩虹表。威胁模型为家庭 LAN + 共享密码，可接受。用户应使用足够强度的密码。

### 5.5 防回环、去重与事件传播（强制）

**主路径：同步写入不得再次外发 + `event_id` 已见集。**

| 优先级 | 机制 | 做法 |
|--------|------|------|
| **P0 强制** | 同步写入不外发 | `write(..., from_sync=true)` 后，剪贴板监听触发的变化 **不得** 再 `ClipboardEvent` 广播 |
| **P0 强制** | `event_id` 已见集 | 收到事件：若 `event_id` 已在 LRU/`HashSet` 中 → **丢弃**；否则记入再处理。容量建议 ≥ 4096 条或按时间滑动（如 10 分钟） |
| **P0 强制** | 本机发出即登记 | 本地产生事件在发送前将 `event_id` 记入已见集 |
| 辅助 | `content_hash` 短窗 | 仅用于损坏检测/UI；**不得**单独作为丢弃用户合法「再次复制相同文本」的唯一依据（短窗防抖可选，窗口宜短，如 ≤1s，且可配置） |
| 辅助 | 写入冷却 | 同步写入后 300–1000ms 内忽略「疑似回声」监听事件（平台兜底） |

**多机传播预期（正确行为）：**

```text
A 复制 → 生成 event_id=E1 → 发给所有已连接节点（B、C）
B 收到 E1 → 已见？否 → 写入剪贴板(from_sync) → 不外发
C 收到 E1 → 同上
（不会出现 B 再把 E1 转发给 C）
```

若未来错误实现导致「转发」：已见集仍保证 C 不重复应用 E1。

### 5.6 可靠性与自愈

| 问题 | 策略 |
|------|------|
| TCP 断开 | dial 方指数退避 + 随机抖动重连 |
| 网络分区恢复 | 抖动避免连接风暴 |
| 监听任务失败 | 看门狗自动重启 listen |
| 剪贴板 API 失败 | 限次重试；不拖死引擎 |
| 重复风暴 | 已见集 + 每秒发送上限 |
| 与远程桌面剪贴板冲突 | 暂停同步开关；文档说明勿多开同步器 |
| 进程退出 | 可选 UDP goodbye；TCP 优雅关闭；shutdown 超时强制退出 |

### 5.7 防火墙与网络准入

| 平台 | 策略 |
|------|------|
| Windows | 首次 bind/listen 可能被 Defender 防火墙拦截。M1：检测「仅本机可连、外机超时」类症状，UI 提示用户允许专用网/专用端口；可选后续用清单请求添加规则（需谨慎提权） |
| Linux | 若 `nftables`/`iptables`/`ufw` 默认拒绝入站，文档说明放行 `tcp/udp 3721`（或配置端口）；应用内提示「检查防火墙」 |
| 通用 | 发现可用但 TCP 失败 → 明确区分「发现成功、数据通道失败」 |

## 6. 剪贴板适配

### 6.1 Windows

- 优先系统事件监听，避免 tight poll
- 文本：`CF_UNICODETEXT` → 规范 UTF-8
- 图片（P1）：优先 PNG 字节；必要时从 DIB 转换
- 文件（P1）：`CF_HDROP` → 读文件内容（受大小限制）或传文件名+内容
- 写入须遵守线程模型；避免长时间占锁

### 6.2 Linux

- 覆盖 **X11 + Wayland**（Debian 桌面常见）
- 文本：`text/plain;charset=utf-8` / UTF8_STRING 等 → 规范 UTF-8
- 图片（P1）：`image/png` 等
- 文件（P1）：`text/uri-list` → 读本地文件
- 无图形会话：首期 **不保证** headless 剪贴板；可后续做「仅网络中继节点」

### 6.3 抽象接口（示意）

```rust
#[async_trait]
trait ClipboardBackend: Send + Sync {
    async fn subscribe(&self) -> Result<ClipboardWatcher>;
    async fn read(&self) -> Result<ClipboardContent>;
    async fn write(&self, content: ClipboardContent, meta: WriteMeta) -> Result<()>;
}
```

`WriteMeta` 必含 `from_sync: bool`（以及可选 `event_id` 便于日志）。

### 6.4 跨平台规范内容（Canonical）

线上与引擎层只认 **规范表示**，平台细节留在 `clipboard` 边界之内。

| kind | 规范载荷 | 规范 mime 示例 | Windows 读 | Linux 读 | Windows 写 | Linux 写 |
|------|----------|----------------|------------|----------|------------|----------|
| text | UTF-8 字节（无 BOM） | `text/plain; charset=utf-8` | CF_UNICODETEXT | text/plain 等 | CF_UNICODETEXT | text/plain |
| image | 编码后的图像字节 | `image/png`（优先）或 `image/jpeg` | PNG/DIB→PNG | image/png 等 | 注册 PNG 或 DIB | image/png |
| file | 文件原始字节 | `application/octet-stream` + `file_name` | HDROP→读文件 | uri-list→读文件 | 可降级为「保存到临时目录 + 文件列表」或仅提供「另存」 | 同左策略 |

**M1 仅强制 text 行。** 图片/文件映射在 M3 实现前补测试矩阵。

文件列表含多个文件时：MVP 可只同步第一个不超过上限的文件，或整组拒绝并提示（实现时二选一写进代码注释与设置说明）。

## 7. UI / UX 设计

### 7.1 技术选型

| 项 | 选择 |
|----|------|
| 框架 | **egui**（+ eframe） |
| 风格 | **现代化：扁平化 + 毛玻璃（Glassmorphism）** |
| 形态 | 主窗口 + 系统托盘（P1） |

### 7.2 视觉原则

- **扁平化**：少阴影层级、清晰几何、统一圆角与间距
- **毛玻璃**：半透明面板 + 观感模糊（半透明 `Color32`、卡片、细高光描边、渐变底）；不强依赖 OS backdrop blur
- **信息密度适中**；动效克制；N100 避免重特效
- **默认深色玻璃风**，浅色可选

### 7.3 色彩与字体（建议规范）

| 令牌 | 建议 |
|------|------|
| 背景 | 深色半透明 `rgba(20,22,28,0.75)` 级 |
| 卡片 | 略亮半透明 + 1px 高光描边 |
| 主色 | 雾蓝 / 青，用于在线与主按钮 |
| 语义色 | 成功 / 警告 / 错误 |
| 字体 | 系统 UI 字体；等宽用于 ID 摘要 |
| 圆角 | 8–12px |
| 间距 | 8px 网格 |

### 7.4 主要界面

1. **主面板 / 历史** — 列表、搜索、点击复制  
2. **设备** — 已连接、附近（UDP）、手动添加  
3. **设置** — 名称、端口、密码、大小上限、开机启动、暂停同步、主题  
4. **状态 / 托盘** — 在线数、错误、暂停  

### 7.5 关键交互文案

| 情况 | 提示 |
|------|------|
| 超限 | 「内容大小 xxx 超过上限 yyy，已取消同步。可在设置中调整上限。」 |
| 鉴权失败 | 「与设备 Z 密码不匹配，请检查共享密码。」 |
| 离线 | 「设备 Z 离线，将自动重连。」 |
| 暂停 | 「同步已暂停（本地复制不会外发）。」 |
| 防火墙 | 「其他设备无法连接本机。请在系统防火墙中允许 OhMyCopy（端口 …）。」 |

### 7.6 历史与性能（P1）

- 内存中以元数据 + 文本摘要 / 缩略图为主；大图按需从磁盘加载  
- 列表虚拟化或分页，避免 N100 上历史过长掉帧

## 8. 配置与存储

### 8.1 配置项（示例）

```toml
# config.toml
config_version = 1              # 迁移用

device_name = "Study-PC"
device_id = "uuid-..."          # 首次生成后固定
tcp_port = 3721
udp_port = 3721

# 密码：见 §8.4，勿写入日志
# 推荐：password 仅存于本地配置文件且权限收紧；或系统密钥环（后续可选）
password = "your-shared-secret"

max_payload_bytes = 10485760    # 10MiB
history_limit = 200
discover_interval_secs = 5
theme = "dark_glass"
auto_start = false
sync_enabled = true
```

### 8.2 存储位置

| 平台 | 路径 |
|------|------|
| Windows | `%APPDATA%/OhMyCopy/` |
| Linux | `~/.config/ohmycopy/` |

目录权限：Linux 配置文件建议 `0600`。

### 8.3 历史数据

- **M3 默认：SQLite** 存元数据与文本；图片 blob 可外置目录并设总容量上限  
- 提供「清空历史」

### 8.4 密码存储策略（已锁定）

| 项 | 决策 |
|----|------|
| M1 | 密码保存在本地配置文件（TOML），**文件权限收紧**；**禁止**写入 tracing/日志 |
| 派生 | 运行时用固定 salt 做 Argon2id，缓存 root_key 于内存 |
| 不存 | 不把 root_key 持久化到磁盘（避免与改密一致性纠缠） |
| 后续可选 | Windows Credential Locker / libsecret，配置中只留「使用密钥环」标志 |
| 改密 | 用户在每台设备手动改同一密码；旧会话断开后用新密码重连 |

### 8.5 配置迁移

- `config_version` 字段；升级时按版本执行迁移函数  
- 未知新版本字段：尽量忽略并保留  
- 破坏性变更：启动时提示备份路径

## 11. 里程碑计划

> 估时为 **单人全职** 参考；业余开发请按 2–3× 放大。v0.1 未给 M1–M4 估时，此处按评审校准。

### M0 — 骨架（约 1 周）

- 工程初始化、日志、配置加载（含 `config_version`）
- egui 空壳 + 深色玻璃主题骨架
- **tokio ↔ egui 桥接按 §4.5 落地**
- CI：`fmt` / `clippy` / `test`（至少 Ubuntu；Windows 尽快补上）

### M1 — 可靠文本同步 MVP（约 3–4 周）

- TCP 监听 + **单连接 tie-break**
- 固定 salt + Argon2id 握手 + AEAD 会话
- Windows + Linux **文本**剪贴板（规范 UTF-8）
- **event_id 已见集** + 同步写入不外发
- 重连退避 + 抖动
- 手动添加设备
- 防火墙失败提示
- **验收**：双机及三机文本同步稳定；1s 内；长提示词；复制相同文本短间隔行为符合预期；无需重启

### M2 — UDP 发现 + 设置 UI（约 1–2 周）

- UDP announce / 设备列表 / goodbye（可选）
- 设置页：密码、端口、名称、大小上限
- 超限提示
- **验收**：新设备可被发现并完成鉴权连接

### M3 — 图片 / 小文件 + 历史（约 3–4 周）

- 图片与文件 kind + §6.4 映射
- SQLite 历史、搜索、缩略图策略
- 托盘、暂停同步
- **验收**：常见截图同步；超限明确提示；内存不因历史暴涨

### M4 — 打磨与发布（约 2–3 周）

- N100 / Debian 性能与内存
- 体积逼近 ≤10MB
- 发布物：Windows 便携 zip + 可选安装器；Linux 静态/半静态二进制（.deb 可选）
- 安装说明、防火墙、与远程桌面共存
- 7 天 soak；代码签名列为可选增强（SmartScreen）

**合计参考：约 10–14 周全职。**

## 13. 风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| Linux 剪贴板碎片化 | 功能不一致 | Backend trait + 测试矩阵 |
| egui 毛玻璃不够真 | 美学 | 半透明设计系统；先扁平现代 |
| 与 RDP/Moonlight 冲突 | 偶发异常 | 暂停开关；勿多开同步器 |
| Argon2 过重 | N100 卡顿 | 调参；缓存 root_key |
| 广播被隔离 | 发现失败 | 手动 IP |
| 体积膨胀 | 超 10MB | feature 裁剪；`cargo bloat` |
| 文件变网盘 | 复杂度 | 严格上限 |
| 错误实现导致事件转发 | 重复写入 | event_id 已见集兜底 |
| TCP 连接风暴 | 恢复期失败 | 重连抖动 |
| 剪贴板写入竞态 | 闪烁/覆盖 | 串行写入队列；后来者胜 |
| 历史图片占内存 | 超 RSS 目标 | 元数据+缩略图；原图磁盘 |
| egui 长列表卡顿 | N100 掉帧 | 虚拟列表/分页 |
| 防火墙拦截 | 「发现得到连不上」 | 检测 + 用户引导 |
| 未签名二进制 | SmartScreen | 文档说明；M4 可选签名 |
| tokio 关闭挂起 | 退不出 | shutdown 超时强制退出 |

## 14. 安全与隐私

- 威胁模型：家庭局域网 + 共享密码；**不宣称**对抗恶意局域网主动攻击者的高保证
- 固定 salt + Argon2id 拉伸；AEAD 保护数据面
- 密码与密钥材料 **禁止**进入日志
- 配置文件权限收紧；历史可清空
- 默认无遥测
- UDP 永不携带密码或剪贴板正文

## 18. v0.2 变更摘要
