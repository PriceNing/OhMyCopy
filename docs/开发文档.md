## 1. 项目概述

### 1.1 一句话定义

**OhMyCopy** 是面向家庭局域网（≤10 台）的轻量级跨设备剪贴板同步工具：Windows / Linux 原生客户端，对等 TCP 同步，UDP 发现，共享密码鉴权，egui 现代化 UI，追求**小体积、低内存、高可靠**。
### 1.2 背景与动机

| 痛点 | 说明 |
|------|------|
| 远程办公 | 通过 Sunshine + Moonlight 连接服务器进行办公与 AI 创作，频繁跨设备复制提示词、图片、文件 |
| 现有方案不可靠 | CrossPaste 等工具存在「复制后无法粘贴、必须重启」类故障，官方修复节奏不可控 |
| 资源占用过高 | Java 方案安装包数百 MB、RAM 占用大、UI 在 N100 等低功耗 CPU 上响应慢 |
| 体积预期 | 作为剪贴板同步工具，**安装/分发体积以 10MB 为上限目标** |

### 1.3 成功标准（验收 KPI）

| 指标 | 目标 |
|------|------|
| 可靠性 | 连续使用 ≥7 天无需因同步故障重启进程 |
| 延迟 | 同局域网文本：复制后 **≤1s** 对端可粘贴 |
| 体积 | Release 单文件 / 安装包 **≤10MB**（strip + LTO） |
| 内存 | 空闲常驻 RSS 目标 **≤50MB**（N100 可接受） |
| CPU | 空闲接近 0；事件驱动，避免高频轮询 |
| 平台 | Windows 10/11 + Linux（含 Debian） |

### 1.4 非目标（明确不做）

- 公网中继、NAT 穿透、云账号体系
- 移动端（iOS / Android）首期不做
- 浏览器扩展 / WebView UI
- 完整「网盘级」大文件传输器
- 与 Moonlight/Sunshine 剪贴板通道深度耦合
- 自动选主、复杂分布式共识
- M1 不引入 PAKE（SPAKE2/CPace 等）；家用共享密码模型足够
- 密码变更自动广播到其他设备（改密后由用户在各设备手动更新）

### 1.5 术语约定

| 用语 | 使用场景 |
|------|----------|
| **设备** | 用户界面、文档叙述（「附近设备」「已连接设备」） |
| **节点** | 协议与实现（`device_id` 标识的网络节点） |
| **载荷** | 帧/事件中的内容字节（不再混用 payload 作正文术语；代码标识符可用 `payload`） |
| **对端** | 已建立会话的另一个节点 |

## 2. 用户与场景

### 2.1 目标用户

- 家庭多 PC 用户（约 6 台，含 Debian）
- 远程桌面办公 + AI 创作（提示词、截图、小文件频繁互拷）
- 对体积、内存、稳定性敏感，可接受功能精简

### 2.2 核心场景

1. **本机 ↔ 远程服务器**：本机复制提示词 → 服务器侧立即粘贴到 AI 工具  
2. **服务器 → 本机**：服务器复制结果/日志 → 本机粘贴  
3. **图片同步**：截图 / 参考图跨设备粘贴（受大小上限约束）  
4. **小文件**：配置范围内的文件随剪贴板同步；超限提示，不静默失败  
5. **多机同时在线**：客厅 / 书房 / 迷你主机等任意子集互相同步  

### 2.3 设备与规模

| 项 | 约束 |
|----|------|
| 规模 | ≤ 10 台 |
| 网络 | 仅局域网 |
| 拓扑 | 对等（P2P），无强制中心服务器 |
| 系统 | Windows、Linux（Debian 等） |

| **P2** | 内容类型过滤 | ⏳ | 未做 |

### 3.2 功能说明

#### 3.2.1 同步内容类型

| 类型 | 行为 |
|------|------|
| 纯文本 / Unicode 文本 | 默认同步；线上 UTF-8 |
| 图片 | PNG 字节 + `image/png`；读端兼容 DIB/DIBV5/PNG/单图 HDROP |
| 文件 | 整包传输 + `file_name`；超过 `max_payload_bytes` 则提示且不发送 |
| 文件夹 | 递归 zip，MIME `application/x-ohmycopy-dir-zip`；收端解压为干净目录名 |
| 其他格式 | 忽略或无法识别时不强制失败 |

#### 3.2.2 大小限制

- 全局配置项：`max_payload_bytes`（**默认 10MiB**，设置页可改；两端宜一致）
- 编码后帧另有实现硬顶（约 512MiB）与按体积计算的 bulk IO 超时
- 超限：本地明确提示，**不发送、不半传**

#### 3.2.3 设备发现与配对

- UDP：周期性 announce（可配置间隔）；载荷**无密码**；魔数 `OMCP`、版本、device_id、名称、tcp_port、caps
- 发现结果仅 UI「附近设备」；**密码鉴权成功**后双方才写入 `clients.json`
- 手动添加 `IP:Port` 触发 trial connect；失败不落盘
- 列表内设备 `auto_connect`（恒为 true）启动后自动重连
- `ignored=true`：可保持 TCP，但静音剪贴板
- 解除配对：发 `Unpair` 并删除本端 clients 项
#### 3.2.4 身份验证

- 全网共享密码（家庭场景）
- 连接建立后挑战-应答；密码 + **协议固定 salt** 派生密钥
- 后续帧均加密；错误密码无法完成握手
- **改密**：用户在每台设备上手动更新同一密码；不提供自动传播

## 4. 系统架构

### 4.1 总体架构

```text
┌─────────────────────────────────────────────────────────┐
│                     OhMyCopy 进程                        │
│  ┌─────────────┐  ┌──────────────┐  ┌────────────────┐  │
│  │  egui UI    │  │  同步引擎     │  │  剪贴板适配层   │  │
│  │  托盘/窗口  │←→│  Engine      │←→│  Win / Linux   │  │
│  └─────────────┘  └──────┬───────┘  └────────────────┘  │
│                          │                               │
│           ┌──────────────┼──────────────┐               │
│           ▼              ▼              ▼               │
│    ┌────────────┐ ┌────────────┐ ┌────────────┐        │
│    │ TCP 会话   │ │ UDP 发现   │ │ 本地存储   │        │
│    │ 单连接/对  │ │ 广播/监听  │ │ 配置/历史  │        │
│    └────────────┘ └────────────┘ └────────────┘        │
└─────────────────────────────────────────────────────────┘
         │ TCP 数据面              │ UDP 发现面
         ▼                         ▼
    其他设备 :port            255.255.255.255 / 子网广播
```

### 4.2 拓扑：对等（P2P）

**不设强制中心服务器。** 每一台运行中的 OhMyCopy：

1. **始终监听** TCP 端口（接受入站连接）
2. **按 tie-break 规则** 决定是否主动连接某一对端
3. 同一二进制、同一种运行模式

| 角色 | 说明 |
|------|------|
| 软件角色 | 每节点对等，无 master 进程 |
| 运维角色 | 常开 Debian 可作「事实上更稳定的节点」，但非特殊服务端 |

全互联：10 台最多约 **45 条** TCP 连接（每对一条），完全可接受。

### 4.3 TCP 连接模型（强制）

**任意两个节点之间最多保留一条 TCP 连接。**

#### 4.3.1 会话唯一性（as-built）

- 连接表主键：`remote_device_id`（live map）；同一对端只保留一条会话。
- 双方均可 trial dial（clients 自动连接 / 手动添加）；**先完成握手者胜**，后到的 socket 丢弃（`already connected`）。
- 方向性 AEAD 密钥仍按 **device_id 大小** 派生 send/recv key，与谁 dial 无关。
- 设计稿中的「仅较小 id dial」不再作为强制行为；以「每对一端一条 live 会话」为准。

#### 4.3.2 重连

- 指数退避 + **全抖动**（例如 `base * 2^n + random(0, base)`），避免网络恢复时连接风暴
- 仅 **dial 职责方** 发起重连；accept 方等待入站

### 4.4 模块划分（as-built）

| 模块 | 路径 | 职责 |
|------|------|------|
| `clipboard` | `src/clipboard.rs` | 剪贴板线程、轮询 watcher、HDROP/DIB/PNG |
| `engine` | `src/engine.rs` | 已见集、sync_guard、大小校验、构造 ClipboardEvent |
| `net::tcp` | `src/net/tcp.rs` | 会话、握手、中继、心跳、Unpair |
| `net::discover` | `src/net/discover.rs` | UDP 发现 |
| `auth` | `src/auth.rs` | Argon2id、会话密钥、AEAD |
| `config` / `clients` | `src/config.rs` / `clients.rs` | `config.json` / `clients.json` |
| `history` / `inbox` | `src/history.rs` / `inbox.rs` | SQLite 历史；收件与 zip |
| `protocol` | `src/protocol.rs` | Message / DiscoverPacket |
| `ui` / `tray` | `src/ui/*` / `tray.rs` | egui、托盘 |
| `app` / `main` | `src/app.rs` / `main.rs` | 生命周期、桥接、headless |
### 4.5 进程内通信与 egui–tokio 桥接（M0 锁定）

**选定模式：双通道 + 请求重绘**

| 方向 | 机制 |
|------|------|
| 引擎 → UI | `tokio::sync::mpsc` 或 `std::sync::mpsc` 推送 `UiEvent`；UI 每帧 `try_recv`，并 `ctx.request_repaint()` / `request_repaint_after` |
| UI → 引擎 | `tokio::sync::mpsc` 发送 `EngineCommand`（改配置、暂停同步、手动添加设备等） |
| 共享只读快照 | 可选 `arc-swap` / `RwLock` 存放「连接列表、最近错误」供 UI 即时读 |

约束：

- **egui 仅在 UI 线程调用**；剪贴板平台 API 若要求特定线程，由 `clipboard` 模块封装，不阻塞 UI 过久
- 单一 `tokio` 运行时在后台线程；`eframe` 主循环在主线程
- Windows 窗口行为：点击 `—` 保留普通窗口并最小化到任务栏（可用 Alt+Tab 找回）；点击 `×` 取消关闭并隐藏到托盘；托盘「退出」才真正结束进程。退出时：发 shutdown → 等网络任务 ≤N 秒 → 超时强制退出

## 5. 网络与协议

### 5.1 传输分层

| 层 | 技术 | 用途 |
|----|------|------|
| 发现面 | **UDP 广播** | 设备发现、在线宣告 |
| 数据面 | **TCP** | 剪贴板内容、可靠有序传输 |
| 安全 | 密码 + 固定 salt 派生密钥 + 会话 AEAD | 鉴权与载荷保护 |

说明：

- 不做 WebSocket
- 首期不上 QUIC
- 公网不在产品范围内

### 5.2 UDP 发现协议

**默认端口**（可配置）：`UDP 3721`

**广播地址**：`255.255.255.255` 或子网定向广播；`SO_REUSEADDR` 按需。

#### 5.2.1 发现包结构（逻辑）

```text
magic:     "OMCP" (4 bytes)
version:   u16
msg_type:  u8   // 1=announce, 2=goodbye
device_id: 16 bytes UUID
tcp_port:  u16
name_len:  u8
name:      UTF-8（显示名）
caps:      u32  // 能力位：text / image / file ...
```

> v0.2：**移除** `fingerprint` 字段（对称密钥模型下无非对称身份可指纹化）。若日后需要展示「配置一致性」，可另加可选 `config_hash`（非安全边界）。

#### 5.2.2 行为

- 启动后立即 announce，之后每 3–10s 一次（可配置，**带抖动**）
- 忽略本机 `device_id` 的回环包
- 超过 N 秒未再收到 announce → UI 标为离线
- **禁止**在 UDP 中传输密码或剪贴板正文

### 5.3 TCP 数据协议

**默认端口**（可配置）：`TCP 3721`（可与 UDP 同号）

#### 5.3.1 连接生命周期

```text
（dial 方）TCP connect  /  （accept 方）accept
  → 交换 device_id + 协议版本（明文 Hello 最小集或首帧）
  → 双连接检测与裁撤（§4.3）
  → Auth 握手（挑战应答 + 派生 session_key）
  → 加密帧会话
  → 心跳 keepalive
  → 优雅 close / 异常后由 dial 方退避重连
```

#### 5.3.2 帧格式

逻辑帧经 **postcard** 序列化后，由 **ChaCha20-Poly1305**（或等价 AEAD）封装上线。

明文逻辑结构示意：

```text
len:     u32   // 密文或 body 长度（实现时定义外层信封）
type:    u16
flags:   u16
body:    bytes // postcard 编码的类型化消息
```

逻辑消息为 **postcard 编码的 `Message` 枚举**（外层 length 前缀 + AEAD）。`FrameType` 数值供文档对照，实现以枚举为准：

| 变体 | 说明 |
|------|------|
| `Hello` | `protocol_version`（**当前 2**）、`device_id`、`device_name`、**`listen_port`** |
| `AuthChallenge` / `AuthResponse` | nonce / proof + 身份 |
| `ClipboardEvent` | 剪贴板条目 |
| `Ack` | 可选 |
| `Ping` / `Pong` | 心跳 |
| `Error` | 错误码与信息 |
| `Unpair` | 主动解配，对端应移除 clients 并断开会话 |

#### 5.3.3 ClipboardEvent body（逻辑）

```text
event_id:     UUID          // 强制；发送方生成；主去重键
source_id:    device_id     // 原始产生该内容的节点
content_hash: 32 bytes      // BLAKE3
kind:         text | image | file
mime:         字符串        // text/plain; charset=utf-8、image/png、
                            // application/octet-stream、application/x-ohmycopy-dir-zip …
created_at:   u64           // unix ms
file_name:    可选字符串    // 文件/文件夹显示名
payload:      bytes         // 规范载荷（见 §6.4）
```

单帧载荷 ≤ 配置的 `max_payload_bytes`（及实现帧硬顶）；大文件依赖 bulk 超时，**暂无分块协议**。
### 5.4 鉴权与加密（已锁定）

| 项 | 决策 |
|----|------|
| 模型 | 全网 **同一共享密码** |
| Salt | **协议级固定 salt**（常量，写入源码/规格，所有节点相同）。**禁止**每设备随机且互不同步的 salt |
| KDF | `Argon2id(password, FIXED_SALT) → root_key`；参数兼顾 N100；**进程内缓存 root_key** |
| 握手 | nonce 挑战 + 证明持有 root_key（HMAC 或 AEAD 密封） |
| 会话 | 派生 `session_key`；业务帧 AEAD 加密 |
| PAKE | **M1–M4 不做** SPAKE2/CPace |
| 失败 | 断开；UI 提示鉴权失败 |

> 说明：固定 salt 使 Argon2 主要起「拉伸密码」作用，而非抗全局彩虹表。威胁模型为家庭 LAN + 共享密码，可接受。用户应使用足够强度的密码。

### 5.5 防回环、去重与事件传播（强制）

**主路径：同步写入不得再次外发 + `event_id` 已见集。**

| 优先级 | 机制 | 做法 |
|--------|------|------|
| **P0 强制** | 同步写入不外发 | `write(..., from_sync=true)` 后，剪贴板监听触发的变化 **不得** 再 `ClipboardEvent` 广播 |
| **P0 强制** | `event_id` 已见集 | 收到事件：若 `event_id` 已在 LRU/`HashSet` 中 → **丢弃**；否则记入再处理。容量建议 ≥ 4096 条或按时间滑动（如 10 分钟） |
| **P0 强制** | 本机发出即登记 | 本地产生事件在发送前将 `event_id` 记入已见集 |
| 辅助 | `content_hash` 短窗 | 仅用于损坏检测/UI；**不得**单独作为丢弃用户合法「再次复制相同文本」的唯一依据（短窗防抖可选，窗口宜短，如 ≤1s，且可配置） |
| 辅助 | 写入冷却 | 同步写入后 300–1000ms 内忽略「疑似回声」监听事件（平台兜底） |

**多机传播（as-built：星型中继 + event_id）：**

```text
A 复制 → event_id=E1 → broadcast 给所有 live 对端
B 收到 E1 → 已见？否 → 记入已见 → 写入剪贴板(from_sync，不因回声再生成新事件)
         → relay 给「除 A 以外」的其他 live 对端（如 C）
C 收到 E1（直接来自 A 或经 B 中继）→ 已见集保证只应用一次
```

因此 **3 台以上不必全互联**：中间节点可中继。全互联时中继可能重复送达，靠 `event_id` 去重。
### 5.6 可靠性与自愈

| 问题 | 策略 |
|------|------|
| TCP 断开 | dial 方指数退避 + 随机抖动重连 |
| 网络分区恢复 | 抖动避免连接风暴 |
| 监听任务失败 | 看门狗自动重启 listen |
| 剪贴板 API 失败 | 限次重试；不拖死引擎 |
| 重复风暴 | 已见集 + 每秒发送上限 |
| 与远程桌面剪贴板冲突 | 暂停同步开关；文档说明勿多开同步器 |
| 进程退出 | 可选 UDP goodbye；TCP 优雅关闭；shutdown 超时强制退出 |

### 5.7 防火墙与网络准入

| 平台 | 策略 |
|------|------|
| Windows | 首次 bind/listen 可能被 Defender 防火墙拦截。M1：检测「仅本机可连、外机超时」类症状，UI 提示用户允许专用网/专用端口；可选后续用清单请求添加规则（需谨慎提权） |
| Linux | 若 `nftables`/`iptables`/`ufw` 默认拒绝入站，文档说明放行 `tcp/udp 3721`（或配置端口）；应用内提示「检查防火墙」 |
| 通用 | 发现可用但 TCP 失败 → 明确区分「发现成功、数据通道失败」 |

## 6. 剪贴板适配

### 6.1 Windows（as-built）

- **专用线程**串行访问剪贴板；watcher **约 400ms 轮询**指纹变化
- 文本：`CF_UNICODETEXT` → UTF-8
- 图片：arboard 图像 + 原生 **CF_DIB / CF_DIBV5 / PNG**；单图路径 HDROP 可提升为 Image
- 文件：`CF_HDROP` → 读文件整包（受 `max_payload_bytes`）
- 文件夹：HDROP 目录 → `inbox::pack_path` 打 zip 后按 File + 特殊 MIME 发送
- 同步写入使用 `from_sync` + **suppress 指纹**（仅抑制匹配内容，避免吞掉后续真实复制）

### 6.2 Linux

- 依赖 arboard / 桌面会话（X11 + Wayland 视环境而定）
- 文本 / 图片 / 文件 URI 尽量映射到同一规范 kind
- **无图形会话**：`--headless` 可作网络节点；剪贴板能力取决于环境（不保证）

### 6.3 实现形态（非 trait 文档示意）

当前为 `ClipboardService`（通道 + 后台线程）+ `spawn_clipboard_watcher`，而非独立 async trait crate。对外 kind 见 `ClipContent::{Text,Image,Files}`。

### 6.4 跨平台规范内容（Canonical）

| kind | 规范载荷 | mime 示例 | 备注 |
|------|----------|-----------|------|
| text | UTF-8 | `text/plain; charset=utf-8` | |
| image | PNG 字节 | `image/png` | |
| file | 原始字节 | `application/octet-stream` 等 + `file_name` | 收端写入 `inbox/<时间戳>/` |
| folder | zip 字节 | `application/x-ohmycopy-dir-zip` | `file_name` 为文件夹名；收端解压 |

多文件一次复制：当前按路径列表依次打包发送（受上限约束）；过大则提示跳过。

## 8. 配置与存储

### 8.1 配置项（as-built：JSON）

主文件 **`config.json`**（`config_version: 2`）。完整示例见 `docs/config-example.json`。

| 字段 | 默认/说明 |
|------|-----------|
| `device_name` / `device_id` | 显示名；UUID 首次生成后固定 |
| `tcp_port` / `udp_port` | 默认 3721 |
| `password` | 共享密码；**禁止打日志** |
| `max_payload_bytes` | 默认 10MiB |
| `history_limit` | 默认 200 |
| `discover_interval_secs` | 默认 5 |
| `theme` | `dark_glass` |
| `auto_start` | 开机启动（平台能力相关） |
| `sync_enabled` | 是否同步 |
| `console` | 是否保留控制台窗口（默认 false） |
| `start_minimized_to_tray` | 启动仅托盘（默认 false）；从托盘恢复后仍是普通任务栏窗口，`—` 到任务栏、`×` 回托盘 |

配对列表 **`clients.json`**：见 `docs/clients-example.json`（含 `ignored`、`source`、`addr`）。

### 8.2 存储位置

| 优先级 | 路径 |
|--------|------|
| **当前** | **`~/.ohmycopy/`**（Windows：`%USERPROFILE%\.ohmycopy`；Linux：`$HOME/.ohmycopy`） |
| 遗留 | exe 同目录便携布局；`%APPDATA%/OhMyCopy/` — 启动时尽量迁移到 `~/.ohmycopy` |

Linux 配置文件权限建议 `0600`。

### 8.3 历史数据

- **SQLite**（`history.db`）存预览与内容；文件/图片路径指向 `inbox/`  
- 列表预览使用短名（如 `[图片] xxx.png`），避免完整 Windows 路径刷屏  
- 提供历史搜索与再复制；inbox 有容量/条数/天数清理策略  

### 8.4 密码存储策略（已锁定）

| 项 | 决策 |
|----|------|
| 现状 | 密码在本地 **`config.json`**；**禁止**写入 tracing/日志 |
| 派生 | 固定协议 salt + Argon2id；root_key 仅内存 |
| 不存 | 不把 root_key 持久化 |
| 后续可选 | 系统密钥环 |
| 改密 | 每台设备手动改同一密码后重连 |

### 8.5 配置迁移

- `config_version`；缺省字段用 serde default 补齐（如 `start_minimized_to_tray`）  
- 遗留 TOML / AppData 路径尝试导入

## 13. 风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
